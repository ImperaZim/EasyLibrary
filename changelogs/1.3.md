# 1.3.0

Released #st September 2024

**For PocketMine-MP 5.18.1**

1.3.0 is a major feature update to EasyLibrary, including support for PocketMine-MP 5.18.1 plugins, bringing a large list of features and improvements over the previous version.

It is **not** compatible with plugins written for version 1.2.0 or lower, and plugins may require code changes to work with it.

**As this changelog is quite large, its accuracy and completeness cannot be guaranteed. Make sure you're looking at the [latest revision](https://github.com/ImperaZim/EasyLibrary/blob/development/changelogs/1.3.md), as it may be amended after relatest**

## Core
- Added a new `/genplugin` command that makes it easier to create plugins based on EasyLibrary as it generates a plugin with a base structure from PluginToolKit.

## Gameplay
- **Avoid Suffocation Fixed:** You will no longer clip through the ground when teleporting to some worlds.

### `imperazim\components\plugin`
#### Highlights 
- Starting from version 1.3, you no longer need to explicitly define the `setPlugin(Plugin $plugin)` method in the `init()` function of your component. The frameworkâ€™s initializer now automatically assigns the plugin instance to the component, making the code more streamlined.
```php
    // For example, previously you would have to define the plugin like this:
    public static function init(PluginToolkit $plugin): array {
      self::setPlugin($plugin);
      // Your Code...
      return [];
    }
```
    
```php
    // Now it is no longer necessary!
    public static function init(PluginToolkit $plugin): array {
      // Your Code...
      return [];
    }
```
- As of version 1.3, this is no longer required. You can now access the plugin directly using `YourComponent::getPlugin()` without needing to manually assign it within the `init()` method.
- In the previous version, it was only possible to register textures using the **PluginResourcePackTrait** trait and passing a texture directory, now it is also possible to register textures passing a CDN using the **PluginCdnResourcePackTrait** trait. See below how to use it:
```php
  use imperazim\components\plugin\PluginToolkit;
  use imperazim\components\plugin\traits\PluginToolkitTrait;
  use imperazim\components\plugin\traits\PluginCdnResourcePackTrait;

  /**
  * Class YourClass
  */
  final class YourClass extends PluginToolkit {
    use PluginToolkitTrait;
    use PluginCdnResourcePackTrait;

    /**
    * Called when the plugin is enabled.
    */
    protected function onEnable(): void {
      self::setInstance(\$this);
      \$this->saveRecursiveResources();
      \$this->registerCdnTextures("cdn.yourlink.com", "textures");
    }
  }
```

### `imperazim\components\utils\ArrayUtils`

The `ArrayUtils` class offers a set of powerful utilities for manipulating arrays, including SQL-like operations such as `SELECT`, `INSERT`, `UPDATE`, and `DELETE`. These methods allow you to query arrays in a manner similar to interacting with databases. Additionally, it provides common array manipulation methods for merging, extracting, and flattening arrays.

#### Key Methods:

#### **Performing SQL-like SELECT Queries:**

- **`select(string $query, array $data): array`**  
  Simulates a SQL `SELECT` query on an array. Supports `WHERE`, `ORDER BY`, and `LIMIT` clauses.
  ```php
  $data = [
      'users' => [
          ['id' => 1, 'name' => 'Alice', 'age' => 25],
          ['id' => 2, 'name' => 'Bob', 'age' => 30],
      ]
  ];
  $result = ArrayUtils::select('SELECT name FROM users WHERE age > 20 ORDER BY name ASC LIMIT 1', $data);
  print_r($result); // Output: [['name' => 'Alice']]
  ```

#### **Performing SQL-like INSERT Queries:**

- **`insert(string $query, array &$data): array`**  
  Simulates a SQL `INSERT` query to insert data into an array.
  ```php
  $data = ['users' => []];
  ArrayUtils::insert("INSERT INTO users (id, name, age) VALUES (1, 'Alice', 25)", $data);
  print_r($data); // Output: [['id' => 1, 'name' => 'Alice', 'age' => 25]]
  ```

#### **Performing SQL-like UPDATE Queries:**

- **`update(string $query, array &$data): array`**  
  Simulates a SQL `UPDATE` query to modify data within an array.
  ```php
  $data = [
      'users' => [
          ['id' => 1, 'name' => 'Alice', 'age' => 25]
      ]
  ];
  ArrayUtils::update("UPDATE users SET age = 26 WHERE name = 'Alice'", $data);
  print_r($data); // Output: [['id' => 1, 'name' => 'Alice', 'age' => 26]]
  ```

#### **Performing SQL-like DELETE Queries:**

- **`delete(string $query, array &$data): array`**  
  Simulates a SQL `DELETE` query to remove data from an array.
  ```php
  $data = [
      'users' => [
          ['id' => 1, 'name' => 'Alice', 'age' => 25]
      ]
  ];
  ArrayUtils::delete("DELETE FROM users WHERE name = 'Alice'", $data);
  print_r($data); // Output: []
  ```

#### **Merging Arrays:**

- **`merge(array $array1, array $array2): array`**  
  Merges two arrays, with values from the second array overwriting the first in case of key conflicts.
  ```php
  $array1 = ['name' => 'Alice', 'age' => 25];
  $array2 = ['age' => 30, 'city' => 'Paris'];
  $result = ArrayUtils::merge($array1, $array2);
  print_r($result); // Output: ['name' => 'Alice', 'age' => 30, 'city' => 'Paris']
  ```

#### **Extracting Values by Key:**

- **`pluck(array $array, string $key): array`**  
  Extracts values for a specified key from an array of associative arrays.
  ```php
  $data = [
      ['id' => 1, 'name' => 'Alice'],
      ['id' => 2, 'name' => 'Bob']
  ];
  $result = ArrayUtils::pluck($data, 'name');
  print_r($result); // Output: ['Alice', 'Bob']
  ```

#### **Flattening Multi-Dimensional Arrays:**

- **`flatten(array $array): array`**  
  Flattens a multi-dimensional array into a single-level array.
  ```php
  $array = [[1, 2], [3, 4], [5]];
  $result = ArrayUtils::flatten($array);
  print_r($result); // Output: [1, 2, 3, 4, 5]
  ```

#### **Grouping Arrays by a Key:**

- **`groupBy(array $array, string $key): array`**  
  Groups an array of associative arrays by a specific key.
  ```php
  $data = [
      ['id' => 1, 'category' => 'A'],
      ['id' => 2, 'category' => 'B'],
      ['id' => 3, 'category' => 'A']
  ];
  $result = ArrayUtils::groupBy($data, 'category');
  print_r($result); 
  // Output:
  // [
  //   'A' => [['id' => 1, 'category' => 'A'], ['id' => 3, 'category' => 'A']],
  //   'B' => [['id' => 2, 'category' => 'B']]
  // ]
  ```

#### **General Query Method:**

- **`query(string $query, array &$data): array|string`**  
  Executes a SQL-like query (`SELECT`, `INSERT`, `UPDATE`, `DELETE`) on an array.
  ```php
  $data = ['users' => [['id' => 1, 'name' => 'Alice', 'age' => 25]]];
  $result = ArrayUtils::query("SELECT name FROM users WHERE age = 25", $data);
  print_r($result); // Output: [['name' => 'Alice']]
  ```

#### Example Usage:
```php
use imperazim\components\utils\ArrayUtils;

// Example 1: Using SELECT query
$data = ['users' => [['id' => 1, 'name' => 'Alice', 'age' => 25]]];
$result = ArrayUtils::query("SELECT name FROM users WHERE age = 25", $data);
print_r($result); // Output: [['name' => 'Alice']]

// Example 2: Merging arrays
$array1 = ['name' => 'Alice', 'age' => 25];
$array2 = ['age' => 30, 'city' => 'Paris'];
$result = ArrayUtils::merge($array1, $array2);
print_r($result); // Output: ['name' => 'Alice', 'age' => 30, 'city' => 'Paris']
```

### `imperazim\components\utils\HttpUtils`

The `HttpUtils` class provides utilities to send HTTP requests (`GET`, `POST`) and parse query parameters from a URL. This class offers easy-to-use methods for making HTTP requests and handling query parameters in URLs.

#### Key Methods:

#### **Sending a GET Request:**

- **`sendGetRequest(string $url, array $params = []): array`**  
  Sends a `GET` request to the specified URL with optional query parameters and returns the response as an associative array.
  
  ```php
  try {
      $response = HttpUtils::sendGetRequest('https://api.example.com/data', ['id' => 123, 'type' => 'json']);
      print_r($response); 
  } catch (Exception $e) {
      echo $e->getMessage();
  }
  ```

  - **Parameters**:
    - `$url` - The URL to send the request to.
    - `$params` - Optional query parameters to append to the URL.
  
  - **Return Value**:  
    Returns the decoded JSON response as an associative array.
  
  - **Throws**:  
    An `Exception` if the request fails (e.g., if `file_get_contents` cannot retrieve data).

#### **Sending a POST Request:**

- **`sendPostRequest(string $url, array $data = []): array`**  
  Sends a `POST` request to the specified URL with optional data to include in the body of the request, and returns the response as an associative array.
  
  ```php
  try {
      $response = HttpUtils::sendPostRequest('https://api.example.com/submit', ['name' => 'Alice', 'age' => 25]);
      print_r($response);
  } catch (Exception $e) {
      echo $e->getMessage();
  }
  ```
  
  - **Parameters**:
    - `$url` - The URL to send the POST request to.
    - `$data` - Optional data to include in the request body.
  
  - **Return Value**:  
    Returns the decoded JSON response as an associative array.
  
  - **Throws**:  
    An `Exception` if the request fails.

#### **Parsing Query Parameters from a URL:**

- **`parseQueryParams(string $url): array`**  
  Extracts and parses query parameters from a given URL into an associative array.
  
  ```php
  $url = 'https://example.com/page?name=Alice&age=25';
  $params = HttpUtils::parseQueryParams($url);
  print_r($params); // Output: ['name' => 'Alice', 'age' => '25']
  ```
  
  - **Parameters**:
    - `$url` - The URL from which to extract query parameters.
  
  - **Return Value**:  
    Returns an associative array containing the parsed query parameters.

#### Example Usage:

```php
use imperazim\components\utils\HttpUtils;

// Example 1: Sending a GET request
try {
    $response = HttpUtils::sendGetRequest('https://api.example.com/users', ['page' => 2, 'limit' => 10]);
    print_r($response);
} catch (Exception $e) {
    echo $e->getMessage();
}

// Example 2: Sending a POST request
try {
    $response = HttpUtils::sendPostRequest('https://api.example.com/users', ['name' => 'John', 'email' => 'john@example.com']);
    print_r($response);
} catch (Exception $e) {
    echo $e->getMessage();
}

// Example 3: Parsing query parameters from a URL
$url = 'https://example.com/search?query=php&sort=desc';
$params = HttpUtils::parseQueryParams($url);
print_r($params); // Output: ['query' => 'php', 'sort' => 'desc']
```

### `imperazim\components\utils\ValidationUtils`

The `ValidationUtils` class provides several utility methods to validate common data types such as emails, phone numbers, URLs, and integers. These methods return boolean values indicating whether the input meets the criteria for being a valid instance of the respective type.

#### Key Methods:

#### **Email Validation:**

- **`isEmail(string $email): bool`**  
  Validates whether the provided string is a valid email address format.

  ```php
  $isValidEmail = ValidationUtils::isEmail('test@example.com');
  var_dump($isValidEmail); // Output: bool(true)
  ```

  - **Parameter**:  
    - `$email` - The email address to validate.
  
  - **Return Value**:  
    Returns `true` if the email is valid, `false` otherwise.

#### **Phone Number Validation:**

- **`isPhoneNumber(string $phone): bool`**  
  Validates whether the provided string is a valid phone number. The phone number format should follow international standards, supporting optional "+" for country codes.

  ```php
  $isValidPhone = ValidationUtils::isPhoneNumber('+123456789');
  var_dump($isValidPhone); // Output: bool(true)
  ```

  - **Parameter**:  
    - `$phone` - The phone number to validate. Should be an international phone number.
  
  - **Return Value**:  
    Returns `true` if the phone number is valid, `false` otherwise.

#### **URL Validation:**

- **`isUrl(string $url): bool`**  
  Validates whether the provided string is a valid URL.

  ```php
  $isValidUrl = ValidationUtils::isUrl('https://example.com');
  var_dump($isValidUrl); // Output: bool(true)
  ```

  - **Parameter**:  
    - `$url` - The URL to validate.
  
  - **Return Value**:  
    Returns `true` if the URL is valid, `false` otherwise.

#### **Integer Validation:**

- **`isInteger(string $value): bool`**  
  Checks if the provided string can be considered a valid integer.

  ```php
  $isValidInteger = ValidationUtils::isInteger('123');
  var_dump($isValidInteger); // Output: bool(true)
  ```

  - **Parameter**:  
    - `$value` - The value to check if it is an integer.
  
  - **Return Value**:  
    Returns `true` if the value is an integer, `false` otherwise.

#### Example Usage:

```php
use imperazim\components\utils\ValidationUtils;

// Example 1: Email validation
$isValidEmail = ValidationUtils::isEmail('user@example.com');
echo $isValidEmail ? "Valid Email" : "Invalid Email"; 

// Example 2: Phone number validation
$isValidPhone = ValidationUtils::isPhoneNumber('+123456789');
echo $isValidPhone ? "Valid Phone Number" : "Invalid Phone Number";

// Example 3: URL validation
$isValidUrl = ValidationUtils::isUrl('https://www.example.com');
echo $isValidUrl ? "Valid URL" : "Invalid URL";

// Example 4: Integer validation
$isValidInteger = ValidationUtils::isInteger('42');
echo $isValidInteger ? "Valid Integer" : "Invalid Integer";
``` 

### `imperazim\components\utils\ResourcePacks`

The `ResourcePacks` class provides functionality to register resource packs for a Minecraft server using reflection to interact with private properties of the resource pack manager. It is designed to manipulate the internal state of the resource pack manager, specifically to add new resource packs dynamically during runtime.

#### Key Method:

#### **Registering a Resource Pack:**

- **`registerPack(string $zipFile): void`**  
  This method registers a zipped resource pack with the server's resource pack manager by accessing and modifying private properties using reflection.

  **Steps:**
  - **Create a new `ZippedResourcePack`**:  
    A new resource pack object is created from the given zipped file path.
  
  - **Use Reflection to Access the Manager's Properties**:  
    The method uses PHP's reflection to access the `resourcePacks`, `uuidList`, and `serverForceResources` properties from the resource pack manager. These are likely private or protected fields in the `ResourcePackManager`.

  - **Add the New Resource Pack**:  
    The method appends the newly created resource pack to the manager's existing `resourcePacks` array and updates the `uuidList` property with the resource pack's UUID.

  - **Force Resource Pack Download**:  
    The server is configured to enforce resource packs by setting the `serverForceResources` property to `true`.

  **Code Example:**

  ```php
  use imperazim\components\utils\ResourcePacks;

  // Path to the zipped resource pack
  $zipFilePath = '/path/to/resource-pack.zip';

  // Register the pack with the server
  try {
      ResourcePacks::registerPack($zipFilePath);
      echo "Resource pack registered successfully!";
  } catch (Exception $e) {
      echo "Failed to register resource pack: " . $e->getMessage();
  }
  ```

  - **Parameter**:  
    - `$zipFile` - The path to the zipped resource pack file.
  
  - **Return Value**:  
    No return value, but throws an exception if an error occurs during registration.

  - **Exception Handling**:  
    If there is any issue during the registration process, the method throws an `Exception`. A `PluginException` is caught separately, logging the error via a `crashdump` for better debugging.

### `imperazim\components\utils\TimeUtils`
The `TimeUtils` class includes a comprehensive set of methods to help with time and date management, from checking if specific times or dates match the current system time, to performing operations like adding or comparing times and dates.

#### Key Methods:

#### **Checking Time Match with Wildcards:**

- **`IsTime(string $time): bool`**  
  Determines if the current time matches the given time in `"HH:MM:SS"` format. You can use the `#` symbol as a wildcard to represent any hour, minute, or second.
  ```php
  if (TimeUtils::IsTime('12:#:00')) {
      echo "It's 12 o'clock!";
  }
  ```

#### **Checking Date Match with Wildcards:**

- **`IsDate(string $date): bool`**  
  Verifies if the current date matches a given date in `"DD/MM"` format. The `#` wildcard can be used for any day or month.
  ```php
  if (TimeUtils::IsDate('25/12')) {
      echo "Merry Christmas!";
  }
  ```

#### **Getting Current Time:**

- **`GetCurrentTime(string $format = 'H:i:s'): string`**  
  Returns the current system time in the specified format (default is `"H:i:s"`).
  ```php
  echo TimeUtils::GetCurrentTime(); // Output: "14:30:25"
  ```

#### **Getting Current Date:**

- **`GetCurrentDate(string $format = 'd/m/Y'): string`**  
  Returns the current system date in the specified format (default is `"d/m/Y"`).
  ```php
  echo TimeUtils::GetCurrentDate(); // Output: "16/09/2024"
  ```

#### **Checking for Weekends:**

- **`IsWeekend(): bool`**  
  Checks if today is a weekend (Saturday or Sunday).
  ```php
  if (TimeUtils::IsWeekend()) {
      echo "It's the weekend!";
  }
  ```

#### **Checking Leap Year:**

- **`IsLeapYear(int $year = null): bool`**  
  Determines if the provided year is a leap year. If no year is provided, the method checks the current year.
  ```php
  if (TimeUtils::IsLeapYear()) {
      echo "This year is a leap year!";
  }
  ```

#### **Adding Time:**

- **`AddTime(string $time, int $hours = 0, int $minutes = 0, int $seconds = 0): string`**  
  Adds a specified number of hours, minutes, or seconds to the given time.
  ```php
  echo TimeUtils::AddTime('10:00:00', 1, 30); // Output: "11:30:00"
  ```

#### **Calculating Time Differences:**

- **`DiffBetweenTimes(string $time1, string $time2): string`**  
  Computes the difference between two times in `"H:i:s"` format and returns it as a formatted string.
  ```php
  echo TimeUtils::DiffBetweenTimes('08:00:00', '10:30:00'); // Output: "02:30:00"
  ```

#### **Checking if a Time Has Passed:**

- **`IsPastTime(string $time): bool`**  
  Verifies if a specified time has already passed today.
  ```php
  if (TimeUtils::IsPastTime('12:00:00')) {
      echo "Noon has passed.";
  }
  ```

#### **Calculating Days Between Dates:**

- **`DaysBetweenDates(string $startDate, string $endDate): int`**  
  Returns the number of days between two given dates in `"d/m/Y"` format.
  ```php
  echo TimeUtils::DaysBetweenDates('01/01/2024', '16/09/2024'); // Output: Number of days
  ```

#### **Getting Start of the Day:**

- **`GetStartOfDay(): string`**  
  Returns the start of the current day in `"00:00:00"` format.
  ```php
  echo TimeUtils::GetStartOfDay(); // Output: "00:00:00"
  ```

#### **Getting End of the Day:**

- **`GetEndOfDay(): string`**  
  Returns the end of the current day in `"23:59:59"` format.
  ```php
  echo TimeUtils::GetEndOfDay(); // Output: "23:59:59"
  ```

#### Example Usage:
  ```php
  use imperazim\components\utils\TimeUtils;
  
  if (TimeUtils::IsWeekend()) {
      echo "Relax, it's the weekend!";
  }
  
  echo "Current time: " . TimeUtils::GetCurrentTime();
  ```

### `imperazim\components\trigger`
**The Trigger class** represents a system of task-based triggers, where a condition is checked and if that condition is met, an action is executed. It is used to manage dynamic tasks in the system, triggering scripts when a certain condition is met.
- **`PerPlayerTrigger:`** To create a trigger that performs an action for each player when a specific condition is met, see below:
   ```php 
    use imperazim\components\trigger\PerPlayerTrigger;
  
    // Define a condition that checks if the player is holding a specific item
    $condition = function(Player $player): bool {
      return $player->getInventory()->getItemInHand()->getId() === Item::DIAMOND_SWORD;
    };
  
    // Define the action to be executed if the condition is met
    $action = function(Player $player): void {
      $player->sendMessage("You are holding a diamond sword!");
    };
  
    // Create a new PerPlayerTrigger
    $perPlayerTrigger = new PerPlayerTrigger($condition, $action);
  ```
- **`GlobalTrigger:`** To create a trigger that performs an action globally when a specific condition is met, see more below:
  ```php 
    use imperazim\components\trigger\GlobalTrigger;
    
    // Define a condition that checks if a specific game event has occurred
    $condition = function(): bool {
      return GameState::isNightTime();
    };
    
    // Define the action to be executed if the condition is met
    $action = function(): void {
      Server::getInstance()->broadcastMessage("Night has fallen!");
    };
    
    // Create a new GlobalTrigger
    $globalTrigger = new GlobalTrigger($condition, $action);
  ```
- **`TriggerManager:`**
  - Registers a new **Trigger** (either **PerPlayer** or **Global**) to be checked at a given interval.</li>
  - The trigger is automatically added to a static list for centralized management and scheduled execution:
  ```php 
    use imperazim\components\trigger\TriggerManager;
    
    // Add the trigger to the Trigger Thread
    TriggerManager::addTrigger($globalTrigger || $perPlayerTrigger);
  ```

### `imperazim\components\cache`
**The Cache** class is responsible for caching data within a system. It allows you to store, retrieve, and invalidate data temporarily or permanently, as well as provide statistics about the cache.
- **Storing and Retrieving Cached Data:** To store data in the cache and retrieve it later:
  ```php 
    use imperazim\components\cache\Cache;

    // Store data in the cache with a key and a time to live (TTL) of 60 seconds
    Cache::put('player_points', 100, 60);
    
    // Retrieve the cached data
    $points = Cache::get('player_points');
    if ($points !== null) {
      echo "Player points: " . $points;
    } else {
      echo "Cache expired or not found.";
    }
  ```
- **Caching Data Without Expiration:** If you want to store data in the cache indefinitely without setting a TTL:
  ```php 
    use imperazim\components\cache\Cache;

    // Store data without expiration
    Cache::put('server_settings', ['difficulty' => 'hard', 'max_players' => 50]);
    
    // Retrieve the cached data
    $settings = Cache::get('server_settings');
    if ($settings !== null) {
      echo "Server settings: " . json_encode($settings);
    }
  ```
